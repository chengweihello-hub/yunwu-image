🧠 先搞懂这些概念

别担心，我用大白话给你讲

🎯 什么是 Span？

想象你在玩赛车游戏，Span 就是游戏里的 checkpoint（检查点）。每到一个地方就打个卡，最后能看到完整的行驶轨迹。

📊 什么是 Telemetry？

Telemetry 就是遥测数据。就像你的智能手表记录心率、步数一样，Telemetry 记录你的 AI 应用在运行时的各种数据。

🤖 什么是 Handler？

Handler 就是大管家。你告诉它"我要记录这个 AI 调用"，它会自动帮你搞定所有事情，不用你操心。

📋 什么是 Invocation？

Invocation 就是调用记录单。你在这个单子上填写这次 AI 调用的信息（比如用了什么模型、多少 Token），然后交给 Handler 处理。

💡 一句话总结

你创建一张调用记录单（Invocation），交给大管家（Handler），大管家会帮你生成检查点（Span），自动记录遥测数据（Telemetry）。

1

获取 Handler 单例（找管家）

# 第一步：导入获取 Handler 的函数
# 这行代码的意思是：从 opentelemetry 工具包里的 genai 模块
# 导入 extended_handler 文件里的 get_extended_telemetry_handler 函数
from opentelemetry.util.genai.extended_handler import get_extended_telemetry_handler

# 第二步：获取 Handler 实例（单例模式）
# 单例模式的意思是：整个程序中只有一个 Handler，不用重复创建
# 就像公司里只有一个总经理，你不用每次都招一个新的
handler = get_extended_telemetry_handler()

# 现在 handler 就是我们的大管家了！
print("管家已就位：", handler)

📖 代码解释

  • from ... import ...：这是 Python 的导入语句，就像从书架上拿一本书下来看
  • get_extended_telemetry_handler()：这是一个函数，调用它会返回一个 Handler 对象
  • handler：这是你给这个大管家起的名字，你可以叫它任何名字（比如 my_handler）
💡 小白提示

如果你看到 ImportError 报错，说明第一步的安装没成功，回去检查一下。

2

创建 Invocation 数据对象（填单子）

# 第一步：导入 AgentInvocation 类
# 这个类就像一个表格模板，你按照模板填写信息
from opentelemetry.util.genai.invocation import AgentInvocation

# 第二步：创建一个 Invocation 对象（填表格）
# 假设我们正在开发一个购物助手 AI Agent
invocation = AgentInvocation(
    # Agent 的名称，相当于这个 AI 的名字
    # 比如你的 AI 叫 "购物小助手"
    agent_name="shopping_assistant",
    
    # Agent 的唯一 ID，相当于身份证号
    # 每个 Agent 实例都有一个独特的 ID
    agent_id="agent_123",
    
    # 输入 Token 数量，即用户说了多少个字（大概）
    # 这里假设用户输入了 150 个 Token
    input_tokens=150,
    
    # 输出 Token 数量，即 AI 回复了多少个字（大概）
    # 这里假设 AI 回复了 80 个 Token
    output_tokens=80,
    
    # 使用的模型名称，就像你用的手机型号
    # 比如 "qwen-turbo"、"gpt-4" 等
    model="qwen-turbo"
)

# 现在 invocation 就是一张填好的记录了！
print("记录单已创建：", invocation)

📖 代码解释

  • AgentInvocation：这是一个类（Class），可以理解为一张表格模板
  • agent_name：Agent 的名字，方便你在监控平台上识别
  • agent_id：唯一标识符，就像身份证号，不能重复
  • input_tokens：输入的 Token 数量，衡量用户输入的大小
  • output_tokens：输出的 Token 数量，衡量 AI 回复的大小
  • model：使用的 AI 模型名称，方便追踪不同模型的性能
💡 小白提示

Token 是什么？可以理解为 AI 的最小文字单位。比如"你好"大概是 2-3 个 Token。你不需要精确计算，AI 框架通常会告诉你。

AgentInvocation 参数说明表

参数名 类型 必填 说明
agent_name string 是 Agent 名称，如 "shopping_assistant"
agent_id string 是 Agent 唯一标识，如 "agent_123"
input_tokens int 否 输入 Token 数量
output_tokens int 否 输出 Token 数量
model string 否 使用的模型名称，如 "qwen-turbo"
3

使用 Context Manager 完成遥测输出（交给管家）

# 第三步：使用 with 语句包裹你的业务代码
# with 语句是 Python 的一个特性，它会自动帮你"收尾"
# 就像你去餐厅吃饭，with 语句会自动帮你买单、收拾桌子

# 开始记录这次 Agent 调用
with handler.start_agent(invocation) as span:
    
    # ========================================
    # 这里放你的业务逻辑代码
    # ========================================
    
    # 假设这里是你调用 AI 模型的代码
    # 比如调用通义千问、ChatGPT 等
    result = agent.execute(user_query)
    
    # 你可以在这里获取 AI 的结果
    print("AI 回复：", result)
    
    # ========================================
    # with 语句会自动帮你完成以下事情：
    # ========================================
    # 1. 自动创建 Span（检查点）
    # 2. 自动挂载 gen_ai.agent.name 等属性
    # 3. 自动记录 Duration（耗时）和 Token Usage Metrics
    # 4. 如果出异常，自动捕获并设置错误状态
    # 5. 自动结束 Span，发送遥测数据

# with 语句结束后，所有数据已经自动发送到观测平台了！
print("遥测数据已发送！")

📖 代码解释

  • with ... as ...：这是 Python 的上下文管理器语法，保证资源正确释放
  • handler.start_agent(invocation)：告诉大管家"我要开始记录一个 Agent 调用了"
  • as span：span 是这次创建的"检查点"对象，你可以用它添加额外信息
  • agent.execute(user_query)：这是你的业务代码，调用 AI 模型
✅ with 语句的魔法：你不需要手动调用任何结束或发送的方法，with 语句会在代码块结束时自动处理一切。即使你的代码抛出了异常，with 也会确保数据正确记录。

# ============================================
# 文件名：genai_observability_demo.py
# 功能：演示如何使用 LoongSuite GenAI Utils 进行可观测性采集
# 作者：你的大名
# ============================================

# --------------------------------------------
# 第一部分：导入所需的库
# --------------------------------------------
# 导入获取 Handler 的函数
# Handler 就像一个大管家，负责帮你记录所有遥测数据
from opentelemetry.util.genai.extended_handler import get_extended_telemetry_handler

# 导入 AgentInvocation 类
# 这是记录单模板，用来填写这次 AI 调用的信息
from opentelemetry.util.genai.invocation import AgentInvocation


# --------------------------------------------
# 第二部分：初始化（只需做一次）
# --------------------------------------------
# 获取 Handler 实例
# 注意：get_extended_telemetry_handler() 使用单例模式
# 无论调用多少次，返回的都是同一个 Handler 对象
handler = get_extended_telemetry_handler()

print("✅ Handler 初始化完成！")


# --------------------------------------------
# 第三部分：封装一个函数，方便复用
# --------------------------------------------
def call_ai_agent(user_query, agent_name="my_agent", model="qwen-turbo"):
    """
        调用 AI Agent 并自动记录遥测数据
        
        参数：
            user_query (str): 用户的输入问题
            agent_name (str): Agent 的名称，用于识别
            model (str): 使用的 AI 模型名称
        
        返回：
            str: AI 的回复内容
        """
    
        # 第一步：创建调用记录单（Invocation）
        # 填写这次 AI 调用的基本信息
    invocation = AgentInvocation(
        agent_name=agent_name,           # Agent 名字
        agent_id="agent_123",           # Agent 唯一 ID（实际项目中应该动态生成）
        input_tokens=150,                # 输入 Token 数（实际项目中应该动态计算）
        output_tokens=80,               # 输出 Token 数（实际项目中应该动态计算）
        model=model                       # 模型名称
    )
    
        # 第二步：使用 with 语句包裹业务逻辑
        # Handler 会自动记录这次调用的所有遥测数据
        with handler.start_agent(invocation) as span:
        
                # ====================================
                # 这里放你的实际业务代码
                # ====================================
        
                # 模拟调用 AI 模型（实际中替换为真实的 API 调用）
                # 比如：调用通义千问、OpenAI、Claude 等
                print(f"正在调用 AI Agent '{agent_name}'...")
        
                # 这里模拟 AI 回复（实际中应该是真实的 AI 调用结果）
        result = f"这是对 '{user_query}' 的回复"
        
                # 可以在 Span 上添加自定义属性（可选）
                # span.set_attribute("custom.key", "value")
        
                print(f"✅ Agent 调用完成！")
        
                return result


# --------------------------------------------
# 第四部分：主程序入口
# --------------------------------------------
if __name__ == "__main__":
    
        print("🚀 开始演示 GenAI Utils 可观测性采集\n")
    
        # 模拟用户提问
    user_question = "今天天气怎么样？"

    response = call_ai_agent(
        user_query=user_question,

        model="qwen-turbo"
    )
    
        print(f"\n📝 最终回复：{response}")

错误处理与异常捕获

with handler.start_agent(invocation) as span:
        try:
                # 调用 AI 模型
        result = agent.execute(user_query)
        
    except Exception as e:
                # 如果出错了，Handler 会自动记录错误信息
                # span 上会自动设置 error.type 和错误状态
                print(f"调用失败：{e}")
                raise  # 重新抛出异常，让 Handler 捕获并记录

📖 代码解释

  • try...except：Python 的错误处理语法，捕获可能发生的异常
  • 即使发生异常，with 语句也会保证 Span 正确结束并发送数据
  • Handler 会自动在 Span 上记录 error.type 等错误信息
+

添加自定义属性

with handler.start_agent(invocation) as span:
        # 在 Span 上添加自定义属性
        # 这些属性会随遥测数据一起发送，方便后续查询和分析
    span.set_attribute("user.id", "user_12345")
    span.set_attribute("user.level", "premium")
    span.set_attribute("request.priority", "high")
    
        # 执行业务逻辑
    result = agent.execute(user_query)

📖 代码解释

  • span.set_attribute(key, value)：给 Span 添加自定义键值对
  • 这些属性会出现在观测平台上，方便你按用户、优先级等维度过滤和分析
  • 属性名建议遵循 domain.name 的命名规范，如 user.id

❓ 常见问题（FAQ）

Q1: 我需要在每个函数里都写 Handler 吗？

不需要！Handler 是单例模式，你在程序开头获取一次，全局都能用。建议把获取 Handler 的代码放在程序入口处，或者封装成一个工具函数。

Q2: 如果我的项目很大，怎么组织代码？

建议创建一个 telemetry.py 文件，把所有与可观测性相关的逻辑封装进去：

# telemetry.py
from opentelemetry.util.genai.extended_handler import get_extended_telemetry_handler
from opentelemetry.util.genai.invocation import AgentInvocation

# 全局 Handler
_handler = None

def get_handler():
    """获取全局 Handler"""
        global _handler
        if _handler is None:
        _handler = get_extended_telemetry_handler()
        return _handler

def create_agent_invocation(**kwargs):
    """创建 AgentInvocation 的工厂函数"""
        return AgentInvocation(**kwargs)

Q3: 数据发送到哪里去了？

默认情况下，Handler 会将遥测数据发送到 OpenTelemetry Collector（一个数据收集器）。你需要配置 Collector 的地址。如果没有 Collector，数据会打印在日志里或丢弃（取决于配置）。

Q4: 对性能有影响吗？

影响非常小！Handler 是异步发送数据的，不会阻塞你的业务代码。就像你发朋友圈，手机会在后台自动上传，不影响你继续刷别的。
